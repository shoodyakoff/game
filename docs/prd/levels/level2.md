# Уровень 2: Стейкхолдеры и требования в продуктовой разработке

## Общее описание
Второй уровень игры погружает игрока в процесс работы со стейкхолдерами и сбора требований для продукта. Игрок выступает в роли Product Manager, которому поручено спланировать новую функциональность командной работы для приложения TaskMaster. Уровень фокусируется на выявлении и анализе заинтересованных сторон, сборе и приоритизации требований, работе с пользователями и анализе конкурентных решений.

## Цели уровня
1. Обучить принципам работы со стейкхолдерами и выявления их потребностей
2. Развить навыки сбора, анализа и приоритизации требований
3. Освоить методы исследования пользователей и анализа конкурентов
4. Научить преобразовывать требования в конкретные product backlog items
5. Сформировать навыки планирования и приоритизации функций продукта

## Структура уровня

### Теоретический блок 1: Основы работы со стейкхолдерами
- **Определение**: Кто такие стейкхолдеры (заинтересованные стороны)
- **Типы стейкхолдеров**: Внутренние (команда, руководство) и внешние (пользователи, клиенты, партнеры)
- **Важность стейкхолдеров**: Почему нельзя игнорировать мнение разных заинтересованных сторон
- **Карта стейкхолдеров**: Как определить всех важных стейкхолдеров и их влияние на проект
- **Методы работы со стейкхолдерами**: Интервью, опросы, наблюдения
- **Потенциальные конфликты**: Как выявлять и разрешать противоречивые требования
- **Интерактивные элементы**:
  * Схема "Матрица влияния и интереса стейкхолдеров"
  * Примеры вопросов для интервью со стейкхолдерами
- Формат: интерактивная инфографика с примерами (не более 3-4 экранов)

### Практический блок 1: Брифинг со стейкхолдерами
- CEO компании ставит задачу: разработать план внедрения новой системы командной работы в приложение TaskMaster
- Ключевые стейкхолдеры делятся своими ожиданиями:
  * CEO: увеличение конверсии и показатели роста использования командной функциональности
  * Дизайнер: создание плавного и интуитивного пользовательского опыта
  * Разработчик: технические ограничения и возможности реализации
  * Маркетолог: потенциал для привлечения новых пользователей
- **Практические задания**:
  * Самостоятельная подготовка списка вопросов для каждого стейкхолдера
  * Проведение интервью (диалог с ИИ в роли стейкхолдеров)
  * Запись выявленных потребностей, ограничений и противоречий
  * Составление карты стейкхолдеров по степени влияния и заинтересованности
- **Оценка ИИ**:
  * Насколько полно выявлены потребности
  * Какие ключевые вопросы были заданы/пропущены
  * Выявлены ли противоречия между стейкхолдерами

### Теоретический блок 2: Сбор и анализ требований
- **Определение**: Что такое требования, их типы (функциональные/нефункциональные)
- **Методы сбора**: Интервью, фокус-группы, анализ конкурентов, пользовательские исследования
- **Документирование требований**: User stories, спецификации, пользовательские сценарии
- **Приоритизация требований**: Критерии и методы (MoSCoW, RICE, Kano)
- **Валидация требований**: Как убедиться, что требования правильно поняты и актуальны
- **Управление изменениями требований**: Как действовать, когда требования меняются
- **Интерактивные элементы**:
  * Шаблоны для документирования разных типов требований
  * Пример матрицы приоритизации
- Формат: интерактивный материал с примерами и шаблонами

### Практический блок 2: Сбор и анализ требований
- Интерактивная доска для работы с требованиями:
  * Сбор требований из интервью со стейкхолдерами
  * Категоризация требований по типам (функциональные, нефункциональные)
  * Выделение конфликтующих требований и их разрешение
  * Формулировка требований в формате user stories
  * Приоритизация требований с помощью метода MoSCoW
- **Форма для заполнения**:
  * Таблица для записи функциональных требований
  * Таблица для нефункциональных требований
  * Форма для user stories (В качестве [роль], я хочу [функция], чтобы [польза])
  * Интерактивная доска для приоритизации MoSCoW
- **Оценка ИИ**:
  * Правильность классификации требований
  * Качество формулировок user stories
  * Обоснованность приоритизации

### Теоретический блок 3: Исследование пользователей
- **Значение исследований**: Почему важно разговаривать напрямую с пользователями
- **Методы пользовательских исследований**: Интервью, опросы, юзабилити-тестирование, A/B тесты
- **Создание профилей пользователей**: Персоны, пользовательские сегменты
- **Методики интервью**: Подготовка вопросов, техники глубинного интервью
- **Анализ пользовательского пути**: Customer Journey Map, точки контакта
- **Выявление болевых точек**: Как определить основные проблемы пользователей
- **Интерактивные элементы**:
  * Шаблоны для подготовки интервью с пользователями
  * Примеры удачных и неудачных вопросов
- Формат: интерактивный материал с примерами и шаблонами

### Практический блок 3: Исследование пользователей
- Возможность проведения виртуальных интервью с 3-5 типажами пользователей:
  * Самостоятельная подготовка вопросов для разных типов пользователей
  * Проведение интервью (диалог с ИИ в роли пользователей)
  * Анализ и группировка потребностей пользователей
  * Выявление основных болевых точек
- **Форма для заполнения**:
  * Текстовое поле для записи вопросов
  * Форма для записи выявленных потребностей
  * Таблица болевых точек с оценкой критичности
  * Шаблон для сегментации пользователей
- **Оценка ИИ**:
  * Качество подготовленных вопросов
  * Глубина выявленных инсайтов
  * Корректность выделения основных проблем

### Теоретический блок 4: Конкурентный анализ
- **Определение и цели**: Зачем нужен анализ конкурентов
- **Типы конкурентов**: Прямые, косвенные, потенциальные
- **Методы анализа**: SWOT, сравнительные таблицы, анализ функций
- **Источники информации**: Где искать данные о конкурентах
- **Сравнительное тестирование**: Как провести юзабилити-тестирование конкурентов
- **Выявление возможностей**: Как найти ниши, не занятые конкурентами
- **Интерактивные элементы**:
  * Шаблон SWOT-анализа
  * Пример сравнительной таблицы конкурентов
- Формат: интерактивный материал с примерами и шаблонами

### Практический блок 4: Анализ конкурентов
- Интерактивная таблица для сравнения аналогичных решений на рынке:
  * Изучение предоставленной информации о 3-4 конкурентах
  * Проведение SWOT-анализа для каждого конкурента
  * Составление сравнительной таблицы функциональности
  * Выявление конкурентных преимуществ каждого решения
  * Определение возможностей дифференциации вашего продукта
- **Форма для заполнения**:
  * Интерактивные SWOT-матрицы
  * Сравнительная таблица функций
  * Поле для записи конкурентных преимуществ
- **Оценка ИИ**:
  * Глубина анализа конкурентов
  * Выявлены ли все ключевые преимущества и недостатки
  * Качество идей по дифференциации

### Теоретический блок 5: Приоритизация и планирование
- **Приоритизация функций**: Основные методики (RICE, Value vs. Effort, Kano)
- **Создание MVP**: Принципы минимально жизнеспособного продукта
- **Планирование итераций**: Как разбить проект на фазы
- **Оценка ресурсов**: Методы оценки трудозатрат и сроков
- **Управление рисками**: Выявление и митигация рисков проекта
- **Определение метрик успеха**: KPI для отслеживания прогресса
- **Интерактивные элементы**:
  * Визуализация матриц приоритизации
  * Примеры планов итеративной разработки
- Формат: интерактивный материал с примерами и шаблонами

### Практический блок 5: Разработка плана и MVP
- Разработка плана проекта:
  * Интерактивная доска для приоритизации требований по методике RICE или MoSCoW
  * Распределение ресурсов и оценка сложности задач
  * Определение временных рамок и вех проекта
  * Учет рисков и разработка плана их минимизации
- Создание MVP (минимально жизнеспособного продукта):
  * Выбор критически важных функций для первого релиза
  * Обоснование выбора с точки зрения ценности для пользователя и бизнеса
  * Планирование итераций и последующих выпусков
- **Интерактивные элементы**:
  * Drag-and-drop интерфейс для создания плана
  * Симуляция ограничений по ресурсам и времени
  * Визуализация влияния решений на ключевые метрики продукта
- **Оценка ИИ**:
  * Качество созданного MVP
  * Сбалансированность распределения ресурсов
  * Реалистичность временных рамок

### Теоретический блок 6: Презентация проекта
- **Структура презентации**: Как выстроить убедительную презентацию
- **Адаптация под аудиторию**: Как учесть интересы разных стейкхолдеров
- **Визуализация данных**: Принципы эффективной визуализации
- **Работа с возражениями**: Как подготовиться к сложным вопросам
- **Техники убеждения**: Storytelling и другие методы
- **Измерение результатов**: Как показать потенциальный успех проекта
- **Интерактивные элементы**:
  * Шаблоны эффективных презентаций
  * Примеры успешных презентаций продуктов
- Формат: интерактивный материал с примерами и шаблонами

### Практический блок 6: Презентация плана
- Подготовка и представление плана заинтересованным сторонам:
  * Автоматическое формирование презентации на основе принятых решений
  * Виртуальная встреча с командой и стейкхолдерами
  * Ответы на вопросы и возражения
  * Корректировка плана на основе обратной связи
- **Финальная оценка**:
  * Реакция стейкхолдеров в зависимости от качества плана
  * Метрики эффективности планирования
  * Анализ сильных и слабых сторон разработанного плана

### Теоретический и практический блок 7: Проверка знаний (Квиз)
- 10 вопросов для проверки знаний по темам:
  * Работа со стейкхолдерами
  * Сбор и анализ требований

  * Конкурентный анализ

## Особенности для разных типов персонажей

### Product Lead (Стратег)
- Бонус к выявлению важности стейкхолдеров
- Повышенная эффективность при стратегическом планировании
- Дополнительные возможности при разработке MVP

### UX Visionary (Дизайнер)
- Лучшее понимание потребностей пользователей
- Бонус при проведении интервью и исследований
- Специальные подсказки при анализе пользовательского опыта конкурентов

### Growth Hacker (DPS)
- Бонус при приоритизации требований по бизнес-ценности
- Улучшенное понимание конкурентных преимуществ
- Специальные подсказки для максимизации метрик роста

### Tech PM (Гибрид)
- Лучшее понимание технических аспектов требований
- Бонус при оценке сложности реализации
- Специальные подсказки о технических ограничениях и возможностях

### Agile Coach (Поддержка)
- Бонус при выявлении и разрешении конфликтов между стейкхолдерами
- Улучшенные навыки фасилитации встреч
- Специальные подсказки по организации итеративного планирования

## Возможные результаты и достижения
1. **Исключительный план (90-100%)**: Максимальное понимание стейкхолдеров, идеальная приоритизация, сбалансированное распределение ресурсов.
2. **Хороший план (70-89%)**: Хорошее понимание потребностей, правильная приоритизация ключевых функций, разумный MVP.
3. **Удовлетворительный план (50-69%)**: Базовое понимание стейкхолдеров, некоторые проблемы с приоритизацией, но работоспособный план.
4. **Требует доработки (<50%)**: Существенные пробелы в понимании стейкхолдеров и требований, слабая приоритизация.
